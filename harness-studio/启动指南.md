# Harness Studio 项目启动指南

## 项目概述

Harness Studio 是一个 AI 开发工作流编排平台，包含三大核心功能：
- **Reverse Engineering**：代码逆向分析与重构规划
- **Multi-Agent Loops**：多智能体工作流编排
- **Quality Dashboard**：AI 代码质量仪表盘

技术栈：
- Next.js 16.2.10（Turbopack）
- React 19.2.7
- TypeScript 5.9.3
- Drizzle ORM + SQLite
- React Flow（流程图）
- Recharts（图表）
- ai-sdk v4（AI 调用）

## 启动步骤

### 1. 进入项目目录

```bash
cd c:\Users\Lenovo\Desktop\AI_project_all\Harness_Studio\harness-studio
```

### 2. 安装依赖

```bash
npm install
```

> **注意**：`better-sqlite3` 是原生模块，安装时需要 Node.js 编译环境。如果安装失败，请确保已安装 `node-gyp` 依赖：
> ```bash
> npm install -g node-gyp
> npm install --global --production windows-build-tools
> ```

### 3. 配置环境变量

项目已预置 `.env.local` 文件，包含 DeepSeek API 密钥。如需修改或添加其他提供商，请编辑该文件：

```bash
# 打开环境变量文件
notepad .env.local
```

可用的 LLM 提供商配置：

| 提供商 | 环境变量 | 说明 |
|--------|----------|------|
| DeepSeek | `DEEPSEEK_API_KEY` | 默认，已配置 |
| OpenAI | `OPENAI_API_KEY` | 可选 |
| Anthropic | `ANTHROPIC_API_KEY` | 可选 |
| Moonshot（Kimi） | `MOONSHOT_API_KEY` | 可选 |
| Qwen（通义千问） | `DASHSCOPE_API_KEY` | 可选 |
| Zhipu（智谱） | `ZHIPU_API_KEY` | 可选 |
| Groq | `GROQ_API_KEY` | 可选 |

> **重要**：`.env.local` 已加入 `.gitignore`，不会被提交到版本库。

### 4. 初始化数据库（首次运行）

```bash
npm run db:push
```

此命令会根据 `lib/db/schema.ts` 创建 SQLite 数据库表。数据库文件位于 `dev.db`。

### 5. 启动开发服务器

```bash
npm run dev
```

启动成功后，访问以下地址：
- **本地地址**：http://localhost:3000
- **局域网地址**：http://192.168.10.103:3000

## 常见问题

### Q1：启动时出现 "params is a Promise" 错误

**原因**：Next.js 16 将动态路由参数改为 Promise，需要显式解包。

**解决方案**：代码已修复，确保使用正确的解包方式：
- 服务器组件：`const { id } = await params`
- 客户端组件：`const { id } = use(params)`

### Q2：LLM 生成时卡住或响应缓慢

**原因**：DeepSeek V4-Pro 模型默认启用 thinking mode，会显著增加响应时间。

**解决方案**：
- 代码已默认禁用 thinking mode
- 默认使用 `deepseek-v4-flash` 模型（响应更快）
- 在设置页面可切换到其他模型

### Q3：数据库连接失败

**原因**：SQLite 数据库文件不存在或权限问题。

**解决方案**：
```bash
npm run db:push
```

### Q4：AI 生成的 Agent 指令为空

**原因**：LLM 提供商未配置或 API 密钥无效。

**解决方案**：
1. 在 `.env.local` 中配置有效的 API 密钥
2. 访问设置页面测试连接
3. 如果使用 DeepSeek，确保密钥以 `sk-` 开头

### Q5：上传代码后页面 404

**原因**：动态路由参数未正确解包（已修复）。

**解决方案**：确保代码已更新到最新版本，动态路由使用 `await params`。

## 技术要点

### Next.js 16 特性
- Turbopack 构建引擎（更快的开发体验）
- 动态路由参数为 Promise，需显式解包
- Server Actions 支持（最大 10MB 请求体）

### AI 提供商切换
- 系统支持自动检测已配置的提供商
- 在设置页面可手动选择偏好提供商
- 提供商配置通过 Cookie 持久化

### 开发模式
- 支持热更新（修改代码自动刷新）
- 无需重启服务器即可看到变更
